- **理由**: ログファイルが非常に大きく（約116MB）、検索に時間がかかっている可能性があります

### 2. RTPパケット受信状況
- **確認結果**: 大量のRTPパケットが受信されている
- **ログ例**:
  ```
  2025-12-27 18:39:27,269 [INFO] [RTP_RECV_RAW] from=('61.213.230.73', 11776), len=172 (pcap)
  ```

### 3. 重要な発見
- **RTP_PEER_FIXED警告が頻発**: RTPピアアドレスが頻繁に切り替わっている
  - `('160.251.170.253', 7168)` ↔ `('61.213.230.73', 11776)`
  - これは、RTPストリームの方向が不安定であることを示している

- **通話終了後のRTPパケット**: 通話終了後もRTPパケットが受信されている
  - `[RTP_SKIP] call_id=in-2025122718385955 already ended`
  - これは、通話終了後もRTPパケットが送信され続けている可能性
